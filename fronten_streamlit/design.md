# Frontend Design Document

## Overview

The frontend is a Streamlit-based web application that provides a professional and engaging chat interface for the AI-powered Ecommerce Support Agent. It enables users to manage orders and receive intelligent, context-aware assistance.

## Components

- **Chat Interface**: A standard message-based UI for real-time interaction with the agent.
- **Order Management Module**: Interfaces for checking order status, requesting cancellations, and creating new orders.
- **Rich Context Display**: A dedicated area or inline message formatting to showcase "fun facts" and product insights generated by the agent.
- **Session Persistence**: Utilizes Streamlit `session_state` and LangGraph `MemorySaver` to maintain conversation history and `thread_id`.
- **Backend Integration**: Connects to the LangGraph orchestration layer, which utilizes Google Gemini (gemini-2.0-flash) and a SQLite database.

## UI/UX Goals

- **Simplicity & Efficiency**: Streamlined chat input and clear feedback for order-related actions.
- **Engagement**: Proactive delivery of interesting product facts based on the order's item type.
- **Reliability**: Immediate visual updates reflecting real-time database status changes (e.g., cancellation success).
- **Professional Persona**: A design aesthetic that reflects a helpful and modern ecommerce support experience.

## Technical Details

- **Frontend Framework**: Streamlit
- **AI Orchestration**: LangGraph
- **Language Model**: Google Gemini (gemini-2.0-flash)
- **State Management**: `st.session_state` and `thread_id` persistence.
- **Database Interaction**: Indirect interaction via the LangGraph agent with the SQLite database (`data/orders.db`).
- **Communication**: Direct invocation of the LangGraph object for processing user queries.
